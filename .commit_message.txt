🚀 Phase 3.7 완료: Structured Logging 구현 (Phase 3 전체 완료!)

Phase 3.7 - Structured Logging:
- Structured Logger (5레벨, JSON 로그, 컨텍스트 관리, 비동기 처리)
- Log Transports (Console, File, HTTP, Memory, Custom)
- Log Formatters (JSON, Text, Colored, Logfmt, Template, Compact)
- Logs API (로그 조회, 고급 쿼리, CSV/Text 내보내기)

Structured Logger:
- 5가지 로그 레벨 (DEBUG, INFO, WARN, ERROR, FATAL)
- 구조화된 로그 데이터 (JSON 기반)
- 로그 컨텍스트 관리 (LogContext.set, with, withAsync)
- 태그 시스템 (로그 분류 및 필터링)
- 성능 측정 (time, timeAsync)
- 자식 로거 생성 (child)
- 비동기 로그 처리 (큐 기반, 100ms 간격)
- Transport 시스템 (플러그형 출력 대상)

Log Transports:
- Console Transport: 콘솔 출력, ANSI 색상 지원
- File Transport: 파일 저장, 자동 로테이션 (크기 기반, 최대 파일 개수)
- HTTP Transport: 원격 서버 전송, 배치 처리 (기본 10개)
- Memory Transport: 메모리 저장, 로그 조회 및 통계 (테스트용)
- Custom Transport: 사용자 정의 로그 처리

Log Formatters:
- JSON Formatter: JSON 형식, Pretty print 지원
- Text Formatter: 읽기 쉬운 텍스트, 타임스탬프 포맷 선택
- Colored Text Formatter: ANSI 색상, 로그 레벨별 색상
- Logfmt Formatter: key=value 쌍 형식
- Template Formatter: 템플릿 기반 커스텀 포맷
- Compact Formatter: 한 줄로 압축된 출력

Logs API:
- GET /api/admin/logs: 로그 조회 (level, message, tags, since, until 필터)
- POST /api/admin/logs/query: 고급 쿼리 (정규식, 태그 AND/OR, 컨텍스트 필터)
- 3가지 포맷 지원 (JSON, Text, CSV)
- 통계 제공 (total, byLevel, byTag, errorCount)
- Permission 기반 접근 제어 (ADMIN_AUDIT_LOGS)

모듈 통합:
- lib/logging/index.ts: 통합 내보내기
- initializeLogging(): 원스톱 초기화
- 헬퍼 함수들 (debug, info, warn, error, fatal)
- 종합 문서화 (README.md, 700+ lines)
